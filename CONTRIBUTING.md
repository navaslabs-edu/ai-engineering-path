# Contributing to AI Engineering Path

This repo is maintained by NavasLabs instructors and improved with help from the community.

## Ways to contribute

- **Fix a typo or broken link** — small, obvious fixes can go straight to a pull request.
- **Suggest an exercise** — open an issue describing the exercise, which session it belongs to, and why it fits.
- **Propose a new or restructured session** — open an issue first to discuss before writing content, so we don't duplicate effort.

## Workflow

1. Fork this repo.
2. Create a branch named `fix/short-description` or `session/NN-short-description`.
3. Make your changes.
4. Open a pull request against `main`, describing what changed and why.
5. An instructor reviews and merges.

## Style

- Keep notes consistent with the existing format: plain language, runnable code examples, a practice section, no filler.
- Practice questions should mix everyday local context with AI-engineering scenarios, matching the existing sessions.
- Every session should build on the one before it — avoid introducing a concept a student hasn't seen yet.

## Code of conduct

Be respectful, be patient with beginners, and keep feedback constructive. This is a learning community first.
